## Default Arguments

sr-lang supports default arguments in function and procedure definitions. This allows you to specify a default value for a parameter, which will be used if the caller does not provide a value for that parameter.

### Basic Usage

You can provide a default value for a parameter by using the `=` operator in the function or procedure signature.

```go
greet :: proc(name: string, greeting: string = "Hello") {
    printf("%s, %s!\n", greeting, name);
};

greet("Alice");      // Output: Hello, Alice!
greet("Bob", "Hi"); // Output: Hi, Bob!
```

### Multiple Default Arguments

You can have multiple parameters with default values. The default values are used from right to left when the caller provides fewer arguments.

```go
configure :: proc(name: string, id: i32 = 0, active: bool = true) {
    printf("Name=%s, ID=%d, Active=%d\n", name, id, active);
};

configure("UserA");                // Output: Name=UserA, ID=0, Active=1
configure("UserB", 10);             // Output: Name=UserB, ID=10, Active=1
configure("UserC", 20, false);      // Output: Name=UserC, ID=20, Active=0
```

Default arguments can make your functions and procedures more flexible and easier to use, especially when you have many parameters that have common default values.
